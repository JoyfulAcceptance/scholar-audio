# Scholar Audio — YouTube Description

Scholar Audio turns dense scholarly papers into structured, chaptered audiobooks without uploading the document to a cloud text-to-speech service.

Drop in a text-based PDF, Markdown file, or plain-text article; choose an installed Mac voice and speaking pace; preview the document; and create a complete `.m4b` audiobook with named chapters. Scholar Audio prepares the text for listening by cleaning extraction artifacts and adding adaptive pacing around new terms, acronyms, numerical findings, dense sentences, and repeated complex results.

Scholar Audio was built for the OpenAI Build Week Education track with Codex and GPT-5.6. Codex helped turn a manual workflow into a tested Python pipeline and local browser interface. GPT-5.6 helped reason through the listening experience and the pacing rules that make scholarly prose easier to follow by ear.

The runtime workflow stays local on the reader's Mac and uses macOS `say`, `pdftotext`, and FFmpeg.
